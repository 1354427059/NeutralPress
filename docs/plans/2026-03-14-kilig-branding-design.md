## 背景

当前站点默认品牌仍是 `NeutralPress`，首页标题、默认标语、默认 logo / favicon，以及部分默认系统页面内容都会露出官方品牌。

仅改代码默认值还不够，因为这个项目会优先读取数据库中的配置；已经初始化过的实例如果数据库里已有旧默认值，页面不会自动跟着源码变化。

## 最终方案

本次采用“默认品牌替换 + 已初始化实例安全补丁”的最小可落地方案：

1. 将默认站点标题、标语、邮件发件人名、SEO 关键词等官方默认品牌替换为 `kilig`。
2. 将首页和浏览器使用的默认 `N` 图标替换为 `K`。
3. 将分享图、manifest、RSS、SEO fallback 里的默认品牌名同步替换为 `kilig`。
4. 在运行期初始化脚本中增加安全补丁：
   - 仅当数据库配置值仍等于官方旧默认值时，才自动更新为 `kilig`
   - 仅当默认系统页内容仍等于官方原始内容时，才替换其中的 `NeutralPress`

## 不主动改动的范围

1. 不批量改 GitHub 仓库地址、上游项目技术名、`NeutralPress Cloud` 等专有名词。
2. 不覆盖用户已经在后台手工修改过的站点标题、标语、页面内容或自定义图片。
3. 不调整与品牌无关的业务逻辑、菜单结构和数据库 schema。

## 生效预期

1. 新部署实例会直接使用 `kilig` 作为默认品牌。
2. 已部署实例在容器启动并执行 runtime init 后，如果仍保留官方默认品牌，也会自动切换到 `kilig`。
3. 若实例已经设置过自己的品牌文案或图片，则不会被这次补丁强行覆盖。
